#### **UX functional mockups**

::::prose-p3
We started work on the UX design stage with workshops, during which we defined expectations regarding the interface and the scope of required functionalities. This allowed us to work out an optimal sitemap along with information architecture. In the next step, we designed a home page prototype, presenting the layout and display method of elements for each of the planned sections. After obtaining mockup acceptance, we created prototypes for the remaining subpages.

UX functional mockups allow for **detailed testing of site functionalities** and checking compliance with requirements already **at an early stage of implementation**. Thanks to this, appropriate corrections can be made, as well as changes in assumptions **without costly work** during further, more advanced stages.

#### **UI user interface design**

::::prose-p3
After completing the UX design stage of the site and obtaining client acceptance, we started work on preparing the graphic design.

We started by choosing appropriate colors and fonts, matching the already existing visual identity of the Mutiny company. Then, based on previously prepared UX mockups, we designed the home page, characterized by a dark theme, high-contrast buttons and unusual shapes. Subpages were designed in grayscale, which further emphasized our client's unconventional activities. In order to maintain consistency and prepare a base for further site development, we created a design system that defines the appearance of all interactive elements (e.g. different button versions, tiles and galleries) and sets the visual framework.

### **Programming implementation**

::::prose-p3
Programming a website is often the most demanding stage of implementation, which requires not only refining the site at the visual level but also taking care of technical issues such as performance or security.

The Mutiny Group website was implemented on the CMS WordPress platform, using the modern WordPress Full Site Editing function. In addition to programming the appearance according to the developed graphic design, we adapted the site to RWD requirements (display on mobile devices, including tablets and different types of phones), extended it with SEO function (enabling the introduction of search engine-friendly content and headers), and also took care of the site's performance by, among others, optimizing graphics and template files. Language versions of the site were introduced using the Multisite function, so that our client maintains full independence in introducing changes for each version. After programming implementation, we handed the site over to the client for content editing.
